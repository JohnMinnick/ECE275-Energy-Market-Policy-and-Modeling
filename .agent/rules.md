# ECE 275 Project Rules

## Code Standards
- All Python code must be well-commented and include docstrings for every function.
- Organize Python scripts with clearly separated sections: Problem Data, Formulation, Solve, Results Analysis, and Plotting.
- Use `numpy`, `scipy`, `matplotlib`, `pandas`, and `pulp` as the primary scientific stack.
- Save all generated figures as PNG files in the same `HWx/` directory for LaTeX inclusion.

## LaTeX Standards
- Use the `IEEEtran` document class (`[journal, onecolumn, 12pt]`).
- Always include the standard preamble packages: `amsmath`, `graphicx`, `listings`, `booktabs`, `geometry`, `xcolor`, `placeins`.
- Use the `mystyle` listing style (defined in the LaTeX report skill template) for all code inclusions.
- Use pastel cell colors (`mygreen`, `myyellow`, `myred`) for table highlighting.
- Include the AI disclaimer in the `\thanks` block of every homework submission.

## File Organization
- Each homework lives in its own `HWn/` folder (e.g., `HW1/`, `HW2/`).
- Inside each `HWn/` folder:
  - `main.tex` — the LaTeX report
  - `solve.py` — the Python solution script (or multiple scripts if needed)
  - `*.png` — any figures generated by the Python scripts
  - `*.pdf` — the assignment PDF (if provided by the instructor)

## Workflow Conventions
- Always activate the `.venv` virtual environment before running Python scripts.
- Compile LaTeX with `pdflatex` (run twice for cross-references).
- Use the `/compile-latex`, `/new-hw`, and `/run-python` workflows for standard operations.
